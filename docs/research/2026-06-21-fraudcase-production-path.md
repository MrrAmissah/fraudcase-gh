## Claude Code orchestration
Set `/goal` to: “Treat FraudCase GH as production software. Close blockers, iterate until App Check, WAF/rate controls, shared limiter, CI/security gates, monitoring, budgets, E2E, abuse, and owner-isolation tests pass; no secrets, no AI-tool provenance leaks, no co-author trailers.”  
Agents: implementer, security, test, docs, release. Prompts:  
Implementer: “Ship only task X; minimal diff; run targeted tests; report risks.”  
Security: “Audit task X for auth, uploads, abuse, secrets, deps; block on high risk.”  
Test: “Run unit, E2E, abuse, owner-isolation; give repros only.”  
Docs: “Update README/docs from merged facts; preserve provenance neutrally.”  
Release: “Verify CI, secrets audit, clean history, rollback note.”

## Workflow and safety gates
Commits: `docs: import research provenance`, `feat: app-check and shared rate store`, `feat: waf quotas and budgets`, `ci: add security gates`, `docs: production readiness`.  
PR gates: `npm test`, `npm run lint`, `npm run build`, env check, E2E, abuse tests, dependency review, secret scan.  
Pre-push audit: `git status --short`; `git log --format=%B | grep -Ei 'Co-authored-by|Claude|ChatGPT|generated by'`; `git ls-files | grep -Ei '(^|/)(\\.env|CLAUDE\\.md|\\.claude/|.*service.*account.*json)'`; `git grep -nI -E 'AIza|BEGIN PRIVATE KEY|private_key|token=|secret=|api_key='`.
